# BuildWithPNJ — Content Strategy

> The playbook for what to create, where to publish, and how to grow
> the BuildWithPNJ brand through strategic content.

---

## 1. Content Mission

> **Teach AI engineering by building real products in public — sharing the code,
> the decisions, the failures, and the wins.**

BuildWithPNJ content exists to:
1. **Attract** — Pull in developers and engineers through valuable technical content
2. **Educate** — Teach practical AI/full-stack engineering with production-grade examples
3. **Build trust** — Demonstrate expertise through transparency, not just credentials
4. **Convert** — Turn readers into followers, subscribers, and eventually customers/clients

## 3. Platform Strategy

### 3.1 Platform Priority

| Priority | Platform | Role | Content Format |
|---|---|---|---|
| **P0** | **Twitter / X** | Primary discovery + engagement | Threads, screenshots, hot takes, updates |
| **P0** | **Blog (buildwithpnj.com)** | Primary long-form, SEO, owned audience | Articles, tutorials, case studies |
| **P1** | **LinkedIn** | Professional visibility, hiring signal | Polished insights, career takes, project updates |
| **P1** | **GitHub** | Code credibility, open-source presence | READMEs, repos, contributions, discussions |
| **P2** | **Dev.to / Hashnode** | Cross-posting for reach, backlinks | Syndicated blog posts (canonical → own blog) |
| **P3** | **YouTube** | Long-form demos, tutorials (future) | Screencasts, walkthroughs, build sessions |

### 3.2 Platform-Specific Rules

#### Twitter / X

| Rule | Example |
|---|---|
| Lead with a hook | "I replaced 4 SaaS apps with one codebase. Here's how 🧵" |
| Use threads for depth (5–12 tweets) | Numbered, each tweet self-contained |
| Include visuals in 60%+ of tweets | Screenshots, diagrams, code snippets |
| Engage 15 min before and after posting | Reply to comments, RT relevant content |
| Use 2–3 hashtags max | #buildinpublic #AIengineering |
| Post at peak times | 9 AM, 12 PM, 6 PM IST / EST |

#### Blog

| Rule | Detail |
|---|---|
| 1,500–3,000 words | Long enough to be valuable, short enough to finish |
| Custom OG image per post | Consistent template with title + brand colors |
| Table of contents | Auto-generated for posts > 1,500 words |
| Code snippets are real | From actual projects, not contrived examples |
| End with CTA | Newsletter signup or related post |
| Cross-post within 48 hours | Dev.to, Hashnode, LinkedIn article |

#### LinkedIn

| Rule | Detail |
|---|---|
| Professional tone | Still human, but less casual than Twitter |
| Use line breaks aggressively | Short paragraphs, 1–2 sentences each |
| Lead with insight, not "I" | "Most AI tutorials teach the wrong thing." > "I wrote a post." |
| Tag relevant people/companies | When genuinely relevant |
| Document carousel posts | Multi-image swipe posts for frameworks/comparisons |

#### GitHub

| Rule | Detail |
|---|---|
| Every repo has a polished README | Description, screenshots, install instructions, architecture |
| Commit messages are clean | Conventional commits: `feat:`, `fix:`, `docs:` |
| Issues are organized | Labels, milestones, project boards |
| Profile README exists | Bio, featured repos, current projects, social links |

## 6. SEO Strategy

### 6.1 Target Keywords

| Keyword Cluster | Priority | Example Queries |
|---|---|---|
| **AI engineering tutorials** | P0 | "fastapi pgvector tutorial", "building RAG pipeline python" |
| **Build in public** | P0 | "build in public developer", "indie hacker AI" |
| **Personal OS / productivity** | P1 | "personal os alternative notion", "developer productivity system" |
| **Full-stack AI** | P1 | "nextjs fastapi template", "full stack AI app architecture" |
| **Developer tools opinions** | P2 | "best vector database 2026", "langchain alternatives" |

### 6.2 SEO Checklist Per Post

- [ ] Title tag includes primary keyword (< 60 chars)
- [ ] Meta description is compelling and includes keyword (< 160 chars)
- [ ] URL slug is short and keyword-rich
- [ ] H1 matches title, H2s include related keywords
- [ ] First paragraph mentions the primary keyword naturally
- [ ] Internal links to 2–3 other posts/projects
- [ ] External links to 1–2 authoritative sources
- [ ] Alt text on all images
- [ ] Custom OG image with title text
- [ ] Canonical URL set (self-referencing or pointing to original)
- [ ] Structured data (BlogPosting JSON-LD)

### 6.3 Backlink Strategy

| Source | Method |
|---|---|
| Dev.to / Hashnode | Cross-post with canonical URL |
| Reddit (r/webdev, r/MachineLearning) | Share genuinely useful posts in relevant threads |
| Hacker News | Submit high-quality tutorials (not self-promotion-heavy) |
| GitHub | Link to blog from repo READMEs |
| Guest posts | Contribute to other blogs with author bio link |

## 7. Newsletter Strategy

### 7.1 Newsletter: "The Build Log"

| Attribute | Detail |
|---|---|
| **Name** | The Build Log |
| **Cadence** | Bi-weekly (every other Tuesday) |
| **Length** | 500–800 words |
| **Tone** | Casual, personal, like a letter from a friend who codes |
| **Tool** | Resend or ConvertKit |

### 7.2 Newsletter Sections

```
Subject: [The Build Log #12] I broke production and learned 3 things

1. 🔨 What I Shipped
   - Brief updates on what was built this sprint

2. 💡 One Thing I Learned
   - A single actionable insight or technique

3. 🔗 Cool Finds
   - 2-3 links to articles, tools, or repos I found useful

4. 📊 By the Numbers
   - Quick metrics: commits, features shipped, subscribers, etc.

Footer: Reply to this email — I read every one.
```

### 7.3 Signup Placements

| Location | Trigger |
|---|---|
| Homepage hero area | Immediate visibility |
| End of every blog post | Reader has consumed value |
| Mid-post CTA | Catch readers who won't finish |
| Dedicated `/newsletter` page | For social media link-in-bio |
| Footer (every page) | Passive capture |
| Exit intent popup | v2 (if needed) |

## 9. Content Production Workflow

```
┌─────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌─────────┐
│  IDEA   │ →  │  DRAFT   │ →  │  REVIEW  │ →  │ PUBLISH  │ →  │ PROMOTE │
│         │    │          │    │          │    │          │    │         │
│ Capture │    │ Write in │    │ Self-edit│    │ Blog +   │    │ Twitter │
│ in notes│    │ Markdown │    │ Read     │    │ OG image │    │ LinkedIn│
│ or idea │    │ in the   │    │ aloud    │    │ SEO pass │    │ Dev.to  │
│ backlog │    │ content/ │    │ Check    │    │ Schedule │    │ Hashnode│
│         │    │ folder   │    │ code     │    │          │    │ Reddit  │
└─────────┘    └──────────┘    └──────────┘    └──────────┘    └─────────┘
     │                                                              │
     └──────────── Recycle & repurpose ◄────────────────────────────┘
```

### Repurposing Chain

```
Blog Post (source of truth)
├── → Twitter thread (condensed, hook-driven)
├── → LinkedIn post (professional angle)
├── → Dev.to / Hashnode (cross-post with canonical)
├── → Newsletter section (excerpt + link)
└── → Future YouTube video (if visual-heavy)
```
