# Copywriting Guide (COPYWRITING_GUIDE.md)

This document establishes the voice, tone, copywriting paradigms, and vocabulary constraints of the **BuildWithPNJ** brand.

---

## 1. Brand Tone & Voice
The platform's voice is **confident, minimal, technical, and precise**. It communicates competence and outcome-driven engineering, avoiding marketing hype and generic buzzwords.

---

## 2. Copywriting Comparisons

| Avoid (Freelancer / Portfolio Tone) | Use (Premium AI Engineering Brand) | Rationale |
| :--- | :--- | :--- |
| "I'm a fullstack developer who likes coding." | "We build production-ready AI systems and automated workspaces." | Signals system capability and professionalism. |
| "I write Python scripts for automation." | "We develop event-driven workflows that automate core operations." | Focuses on systemic value. |
| "I use AI to build websites." | "We integrate local LLM frameworks and secure semantic search pipelines." | Specifies technical architectures. |
| "Check out my personal blog here." | "Explore the Engineering Journal for systems walkthroughs." | Matches the engineering brand style. |

---

## 3. Writing Rules
- **Outcome-First**: Lead with the result (e.g. latency reduced by 40%) before explaining the stack.
- **Short Sentences**: Keep paragraphs under 3 sentences to maintain visual space.
- **Strict Grammar**: Ensure technical configurations are named correctly (e.g. Next.js, FastAPI, PostgreSQL, Supabase).
