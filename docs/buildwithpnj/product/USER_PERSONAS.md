# BuildWithPNJ — User Personas

> Who are we building for? This document defines the core audience segments
> for both the BuildWithPNJ brand website and the products we ship.

---

## 1. Primary Personas

### 1.1 🧑‍💻 Dev-Curious Darshan — The Fellow Builder

> _"I want to learn from someone who actually ships, not just tweets about it."_

| Attribute | Detail |
|---|---|
| **Age** | 22–32 |
| **Role** | Junior–Mid Software Engineer, CS Student, Self-taught Developer |
| **Location** | India, Southeast Asia, Global (English-speaking) |
| **Tech Comfort** | High — writes code daily, uses GitHub, follows tech Twitter |
| **Platforms** | Twitter/X, GitHub, Dev.to, YouTube, LinkedIn |

**Goals:**
- Learn practical AI engineering from real-world projects (not toy demos)
- Find inspiration from someone building in public with a similar background
- Discover tools, frameworks, and architectural patterns worth adopting
- Build their own portfolio and personal brand

**Pain Points:**
- Drowning in theoretical AI content; wants hands-on, production-grade examples
- Intimidated by polished tech influencers who hide the messy parts
- Doesn't know where to start with full-stack AI products
- Struggles to bridge the gap between tutorials and real products

**What They Want from BuildWithPNJ:**
- Transparent build logs showing the real process (wins + failures)
- Code they can read, fork, and learn from
- Architecture decisions explained with rationale
- A roadmap they can adapt for their own journey

**Content That Resonates:**
- "How I built X from scratch" blog posts with code
- Twitter threads breaking down architecture decisions
- GitHub repos with clean READMEs
- Before/after screenshots of UI evolution

---

### 1.2 💼 Hiring-Manager Hema — The Evaluator

> _"Show me you can ship production software, not just solve LeetCode."_

| Attribute | Detail |
|---|---|
| **Age** | 30–45 |
| **Role** | Engineering Manager, Tech Lead, CTO at startup |
| **Location** | Global — US, India, Europe |
| **Tech Comfort** | High — technical background, evaluates engineers daily |
| **Platforms** | LinkedIn, GitHub, Company career pages |

**Goals:**
- Find strong full-stack / AI engineers who can ship independently
- Assess technical depth beyond a resume
- Verify someone can build real systems, not just follow tutorials
- Gauge communication skills and builder mindset

**Pain Points:**
- Portfolios are usually dead links or todo-app clones
- Hard to tell if someone actually built what they claim
- Resumes don't show architectural thinking or taste
- Interview performance ≠ on-the-job performance

**What They Want from BuildWithPNJ:**
- A polished portfolio with live demos and source code
- Evidence of end-to-end ownership (design → build → deploy → iterate)
- Clear technical writing that demonstrates depth
- Professional presentation that signals senior-level thinking

**Content That Resonates:**
- Project deep-dives with architecture diagrams
- Clean, well-documented GitHub repositories
- A polished About page with clear narrative
- Blog posts showing problem-solving process

---

### 1.3 🤝 Collab-Ready Chirag — The Potential Partner

> _"I'm building something in AI too — maybe we should work together."_

| Attribute | Detail |
|---|---|
| **Age** | 25–40 |
| **Role** | Indie hacker, Startup founder, Freelance engineer, Content creator |
| **Location** | Global |
| **Tech Comfort** | High — building their own products |
| **Platforms** | Twitter/X, LinkedIn, Indie Hackers, Discord communities |

**Goals:**
- Find collaborators for AI projects (co-building, open source)
- Cross-promote with complementary builders
- Learn from someone shipping in a similar domain
- Explore freelance / consulting partnerships

**Pain Points:**
- Hard to find serious builders (vs. all-talk accounts)
- Most "build in public" accounts fizzle after 2 weeks
- Needs to see consistency and quality before reaching out
- Collaboration requires trust — portfolio is the proof

**What They Want from BuildWithPNJ:**
- Proof of consistent shipping over time (changelog, commit history)
- Contact info and openness to collaboration
- Complementary skills they can leverage
- Shared values (transparency, quality, AI-native)

**Content That Resonates:**
- Monthly progress updates with metrics
- Open-source repos they can contribute to
- Hot takes on AI tools and industry trends
- Direct "open to collabs" messaging

## 2. Secondary Personas

### 2.1 📱 Casual-Consumer Kavya — The Curious Visitor

> _"I saw a cool tweet and clicked through. Impress me in 5 seconds."_

| Attribute | Detail |
|---|---|
| **Age** | 20–35 |
| **Role** | Tech-adjacent professional, Product Manager, Designer, Student |
| **Tech Comfort** | Medium — understands tech at a high level, doesn't code daily |
| **Entry Point** | Social media link, Google search, referral |

**Goals:**
- Understand what BuildWithPNJ is in < 10 seconds
- See something visually impressive
- Maybe sign up for the newsletter if the content is interesting

**What They Need:**
- Crystal-clear hero section with immediate value prop
- Visual polish that signals credibility
- Easy navigation — don't make them think
- A "what's this about?" answer above the fold

---

### 2.2 🏢 Client-Ready Ravi — The Buyer

> _"I need someone who can build AI-powered tools for my business."_

| Attribute | Detail |
|---|---|
| **Age** | 30–50 |
| **Role** | Startup founder, Small business owner, Product lead |
| **Tech Comfort** | Low-Medium — knows what they want, not how to build it |
| **Entry Point** | LinkedIn, Google search, referral |

**Goals:**
- Find a reliable AI engineer for consulting / contract work
- See proof of past work (portfolio, case studies)
- Understand pricing and availability
- Get in touch quickly

**What They Need:**
- Services section or clear "available for hire" signal
- Case studies with business outcomes (not just tech specs)
- Easy contact form with fast response
- Professional credibility signals (testimonials, client logos)

## 5. Design Implications

| Persona Need | Design Decision |
|---|---|
| Darshan wants to learn | Blog with syntax highlighting, code blocks, copy buttons |
| Hema wants proof | Project pages with screenshots, architecture diagrams, live links |
| Chirag wants to connect | Prominent social links, "Open to collabs" badge, contact page |
| Kavya needs instant clarity | Hero section answers "who/what/why" in < 5 seconds |
| Ravi needs to trust | Testimonials, professional tone, case study outcomes |
